# Угадай по смайликам

Десктопная викторина на C# и WinForms: на экране - это различный набор эмодзи, по ним нужно угадать текущий фильм.
Данные подготовлены из двух открытых датасетов (Kaggle + IMDb) с помощью Python-скрипта.

## Этап 1: подготовка данных

Скрипт `prepare_data.py` берет два открытых датасета и собирает таблицу "русское название, эмодзи":

| Источник | Что берём |
|---|---|
| [Movies to Emojis (Kaggle)](https://www.kaggle.com/datasets/tarundalal/movies-to-emojis-dataset) | английское название + эмодзи |
| [IMDb datasets](https://datasets.imdbws.com/) (`title.basics.tsv`, `title.akas.tsv`) | код фильма + русское название |

Связываем по идентификатору `tconst` (вид `tt0110357`).

**Результат:** `movies_ru_emoji.tsv` - 169 фильмов с русскими названиями и эмодзи.

**Потери по шагам:**
- Исходных фильмов с эмодзи: 253
- Найдено в IMDb: 227 (−26, названия записаны иначе или фильма нет в базе)
- С русским переводом: 169 (−58, перевод не занесён в IMDb)
- Основная причина потерь — отсутствие русского перевода в IMDb (69% всех потерь)

## Этап 2: приложение

**Стек:** C# · .NET 10 · WinForms · SQLite · EF Core

**Что умеет:**

- Главное меню с выбором режима
- Игра: случайный вопрос (эмодзи), 4 варианта ответа, счет, показ правильного ответа
- Режим администратора: добавить свой вопрос (эмодзи + ответ + категория), он сразу попадает в игру и сохраняется после перезапуска
- Валидация ввода: пустые поля не добавляются
- База создается и заполняется автоматически при первом запуске

## Что реализовано

В рамках проекта реализованы следующие возможности:

* подготовка данных из двух открытых источников (Kaggle и IMDb) с использованием Python и библиотеки pandas;
* автоматическое создание базы данных SQLite и её первоначальное заполнение при первом запуске приложения;
* главное меню с выбором игрового режима и режима администратора;
* игровой режим со случайным выбором вопроса, четырьмя вариантами ответа, подсчётом счёта и отображением правильного ответа;
* режим администратора для добавления новых вопросов (эмодзи, ответ, категория);
* проверка корректности ввода: запрещено добавление записей с пустыми полями и дублирующимися ответами;
* сохранение всех пользовательских вопросов в базе данных между запусками программы;
* обработка ошибок запуска приложения с выводом понятного сообщения пользователю;
* разделение логики приложения: работа с базой данных вынесена в отдельный класс `Db`, пользовательский интерфейс не обращается к базе напрямую.
* фильтр вопросов по категории и настройка длины раунда / времени на ответ
  в главном меню;
* обратный отсчёт времени на каждый вопрос, автоматический переход
  к следующему при истечении времени;
* защита от повторения одного и того же вопроса два раза подряд;
* экран результата по итогам раунда (счёт из общего числа вопросов).

**Архитектура:** работа с базой вынесена в отдельный класс `Db`, формы не обращаются к базе напрямую.
